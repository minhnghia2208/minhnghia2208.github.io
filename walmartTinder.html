Features:

• Real-time interaction:


There are several ways to achieve real-time interaction:

1. We can periodically send API request to server and update information accordingly.

    • Pros: Very simple to implement.

    • Cons: Not resourceful and traffic heavy for both server and users' devices.

2. We can periodically check for new information from Back-end and only send response if there is new data.

    • Pros: Better for user's device as it doesn't need to handle a barrage of responses

    • Cons: Still not resourceful and traffic heavy for server.

3. We can use SignalR which handle WebSockets.

    • Pros: Put less strain on server and devices

    • Cons: Compare to other options, there is none



• Azure Cognitive Services:


Our goal is to improve UX by incorporating Azure Cognitive Services.

Everytime a user like another user; Walmart Tinder will send the user's preference to Azure Cognitive Service which is a Deep Learning Algorithm used to process images. The App will then use predictions from Azure to improve UX.
